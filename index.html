Scandium

Scandium was a project I undertoook in 2022 while I was still looking for a job for the summer. Instead of doing nothing while waiting for callbacks, I decided, hey, why not practice my C++ and learn something new? And thus, I started learning Vulkan and made barebones game engine from scratch. The result: I learned about the graphics pipeline and how it works "under the hood", I got a new major linear algebra refresher along with new insights, more advanced C++ techniques and better memory management patterns, and a high-quality 60 FPS rotating cube. Of course, adding sprites and 3D models would be relatively easy with tinyobjloader and such, but manually creating vertices is more fun and challenging.

I chose the name "scandium" for a simple reason: the chemical symbol is Sc, which corresponds to my initials.

Handwritten LaTeX

In 2021, I was still studying at McGill towards my physics degree. I initially tried taking notes digitally, both handwritten and typed, but nothing beat paper when it comes to helping me remember stuff. The main issue was portability: moving paper around can be a real pain sometimes. So I thought, why not digitize them first and then turn them into LaTeX so I can rearrange my notes? Armed with Python, TensorFlow, OpenCV2 and some free time, I first built a script that would take digitized images, apply image processing in order to get rid of imperfections, random scan lines and paper lines, and then apply bounding boxes to each character. These characters were then fed to the AI model which was trained on a few hundred handwritten characters from my own handwriting (to the left is an example). The LaTeX code was then output to the GUI. While the confidence wasn't exceptional (~80%), it was a great learning experience that made me understand how machine learning works "under the hood" through the use of linear algebra. It wouldve have been possible to end up with better latin character recognition if I had used the MNIST database and other such sets.
